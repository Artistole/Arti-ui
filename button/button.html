Button
The button is a simple yet elegant clickable interface to let the user interact with the website.
Contained Button
Contained buttons are of high-emphasis, distinguished by their use of elevation and fill. They contain actions that are primary to your app.
Code Snippet:
Use class btn for button styling and use class btn-primary or btn-secondary or btn-disabled as per the requirement.
Outlined Buttons
Outlined buttons are medium-emphasis buttons hence which is why they can be smaller than the primary emphasis buttons. They contain actions that are important, but arenâ€™t the primary action in an app.
Code snippet:
Use class btn and class btn-outline or btn-outline-small as per the requirement.
Text Buttons
Text buttons are typically used for less-pronounced actions, including those located in dialogs or in cards.
Code snippet:
Use class btn and class btn-text.
Buttons with icons
Sometimes you might want to have icons for certain button to enhance the UX of the application as we recognize logos more easily than plain text.
Code snippet:
Use class btn and class btn-icon-primary or btn-icon-secondary as per the requirement and for icon use class icon.
Floating action button
Icons are also appropriate for toggle buttons that allow a single choice to be selected or deselected, such as adding or removing a star to an item.
Code snippet:
Use class btn and class btn-float-primary or btn-float-secondary as per the requirement and no styling is required for icon.
